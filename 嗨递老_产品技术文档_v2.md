# 嗨递老：具身智能火锅店无障碍服务系统

### Product & Technical Document v2.0 · 火锅场景专版

> **品牌含义**：
> - **嗨** — 海底捞式的快乐服务文化（"嗨"是年轻化的问候）
> - **递** — 机械臂作为物理延伸，"递"到老人手中的核心动作
> - **老** — 对老年群体的致敬，不是"老龄"的负担感，而是"老朋友"的亲切感
>
> 英文名：**HiDilao** — 既致敬 Haidilao，又暗含 "Hi, Dilao（递老）" 的双关。

## 一、品牌定义

### 1.1 核心价值主张

> **"海底捞把服务做到了极致，嗨递老让服务弯得下腰。"**

市面上绝大多数"适老化"产品的逻辑是错的——它们默认用户需要被"教育"来适应技术。嗨递老反转这个命题：

**不是让人去适应机器，而是让机器主动弯下腰来服务人。**

在每一家火锅店，无需改造任何固定设施，只需一套桌面固定的协同系统，就能让无障碍就餐成为现实——不是靠用户的努力，而是靠系统的主动适配。

### 1.2 差异化定位

| 维度 | 说明 |
|------|------|
| **赛道** | 餐饮具身智能 / 火锅场景服务机器人 |
| **目标用户** | 65 岁以上老年人、轮椅用户、视障人士、家庭聚餐场景中的长辈 |
| **差异化** | 不是替代服务员，而是让服务员的能力 **物理延伸**——一个服务员 + 一台嗨递老 = 同时服务 8 桌的极致关怀 |
| **技术底座** | AGX NERO 7-DOF 机械臂 + ROS2 Jazzy + AGX Gripper |

## 二、市场洞察

### 2.1 火锅餐饮行业痛点

火锅是中国餐饮第一大赛道（2025 年市场规模超 6000 亿元），海底捞以"极致服务"闻名全球。但火锅场景中存在三个被忽视的系统性盲区：

**痛点一：屏幕距离悖论**

送餐机器人的屏幕固定在大约 1.0-1.2 米高度——这是站立成年人的舒适视线高度。但对轮椅用户（视线高度约 1.0-1.1 米）和驼背老人（视线高度约 0.9-1.0 米）来说，他们需要抬头 30-45° 仰视屏幕，根本无法阅读菜单。这不是 UI 问题，是**物理问题**。软件解决不了高度。

**痛点二：服务员的"最后一米"困境**

海底捞服务员以"蹲下来为顾客点单"闻名。但一个服务员同时负责 4-8 桌，高峰时段无法为每一桌老人蹲下服务。服务意愿有上限——服务员的膝盖只能弯那么多次。当隔壁桌催加汤时，老人那一桌的蹲式服务就被牺牲了。

**痛点三：生日经济的降级体验**

海底捞的生日庆祝是一大特色。但当前方式是服务员举着灯牌站在桌旁唱生日歌——老人坐在角落，灯牌举在 1.6 米高度，他看到的只有服务员的下巴和灯牌的底部。本应是"被珍视"的时刻，物理上却变成了"被俯视"的体验。

### 2.2 老年人用餐体验盲区

| 盲区 | 现状 | 嗨递老的回答 |
|------|------|------------|
| 看不清菜单 | 扫码点餐字体小、反光、需要多次缩放 | 大字版菜单推送到眼前 50cm |
| 不会操作 | 智能手机操作门槛高 | 一键切换，无需学习 |
| 忌口被忽视 | 服务员口头询问，容易遗漏 | 健康标签可视化 |
| 被边缘化 | 家庭聚餐中老人常常被动等待 | 机械臂主动"蹲下"，成为关注焦点 |
| 生日尴尬 | 被灯牌围着唱生日歌，心理压力大 | 机械臂笨拙但认真的舞蹈，温暖而非压迫 |

### 2.3 竞品对比矩阵

| 维度 | 海底捞送餐机器人 | 普通点餐平板 | 人工服务员 | **嗨递老** |
|------|-----------------|-------------|-----------|----------|
| 屏幕高度 | 固定 1.0-1.2m | 桌面平放 | 手持/蹲下 | **自适应 0.6-1.2m** |
| 老年人适配 | 字体偏小 | 无大字模式 | 靠服务员蹲下 | **一键大字版 + 暖色高对比** |
| 生日体验 | 无 | 无 | 人举灯牌 | **机械臂舞蹈 + 屏幕庆生** |
| 物理递送 | 仅送餐盘 | 无 | 服务员手递 | **屏幕递到眼前 50cm** |
| 24/7 一致性 | ✅ | ✅ | ❌ 疲劳 / 情绪波动 | ✅ |
| 服务温度 | 机械冰冷 | 冰冷 | 温暖但不稳定 | **弯腰的温暖 × 机器的稳定** |

## 四、功能模块分解

### 4.1 屏幕模块

#### MVP：基础菜单 + 大字版

| 特性 | 基础版 | 大字版 |
|------|--------|--------|
| 字号 | 16-18px | **28-36px** |
| 每屏菜品数 | 8-10 道 | **4-6 道** |
| 背景色 | 白色 | **暖黄色 (#FFF8E7)** |
| 对比度 | 标准 | **高对比 (WCAG AA+)** |
| 图片尺寸 | 80×80px | **160×160px** |
| 切换方式 | — | 上位机一键切换 |
| 技术实现 | HTML/CSS/JS 静态页面 | 同技术栈，独立 CSS |

#### 进阶：菜品专注模式

- 全屏轮播：每道菜单独占据整屏，仅显示菜品摄影图 + 菜名
- 健康标签：低盐（绿）、低糖（蓝）、清淡（青）、高蛋白（金）
- 视觉滤镜：菜品图片统一调色为暖色调，增加食欲感
- 轮播速度：8 秒/道，可暂停

### 4.2 机械臂模块（AGX NERO 7-DOF）

#### 硬件规格

| 参数 | 值 |
|------|-----|
| 型号 | AGX NERO |
| 自由度 | **7-DOF**（joint1 ~ joint7） |
| 通信方式 | CAN Bus（can0，1 Mbps） |
| 末端执行器 | AGX Gripper（夹爪）或屏幕固定支架 |
| 控制方式 | ROS2 JointTrajectoryController |
| 软件栈 | ROS2 Jazzy + agx_arm_ctrl + agx_arm_moveit |

#### 关节参数

| Joint | 名称 | 最小 (rad) | 最大 (rad) | 功能描述 |
|-------|------|-----------|-----------|---------|
| joint1 | 底座旋转 | -2.705 | +2.705 | 左右朝向调整 |
| joint2 | 肩部俯仰 | -1.745 | +1.745 | 大臂前后伸展 |
| joint3 | 肘部俯仰 | -2.758 | +2.758 | 小臂折叠角度 |
| joint4 | 腕部俯仰 | -1.012 | +2.147 | 末端倾角控制 |
| joint5 | 腕部翻滚 | -2.758 | +2.758 | 末端翻转 |
| joint6 | 末端偏航 | -0.733 | +0.960 | 精细指向 |
| joint7 | 末端旋转 | -1.571 | +1.571 | 屏幕 / 夹爪定向 |

#### MVP 功能详述

**① 固定舒适坐标**

预设一个经过调试的 7 关节目标位姿，使屏幕处于最舒适的观看位置：

```
# 目标示例（需实际硬件调试确认）
comfort_pose = [0.0, -0.5, 0.3, 0.8, 0.0, 0.3, 0.0]
# joint1=0.0     → 正前方
# joint2=-0.5    → 肩微前倾
# joint3=0.3     → 肘微收
# joint4=0.8     → 腕倾约 15°（屏幕面向用户）
# joint5=0.0     → 无翻滚
# joint6=0.3     → 微调指向
# joint7=0.0     → 屏幕水平
```

**② 生日舞蹈律动**

预录关键帧序列，随海底捞生日歌节奏执行：

```
舞蹈序列结构（约 30 秒）：
├── 前奏 0-5s：机械臂从舒适位微抬起，准备
├── 主歌 5-15s：左右摆动（joint1 小幅正弦运动）
├── 副歌 15-25s：划圆弧轨迹（joint2+joint3 配合）
├── 高潮 25-30s：末端点头+旋转（joint4+joint7）
└── 结束：回到舒适坐标
```

关键帧以 `JointTrajectory` 消息预录制，通过 ROS2 `JointTrajectoryController` 回放。速度百分比设为 20-30% 以确保安全和平滑。

**③ 上位机控制接口**

通过 ROS2 服务/话题控制：

| 操作 | ROS2 接口 | 说明 |
|------|----------|------|
| 启动/上电 | `/enable_agx_arm` 服务 | 使能机械臂 |
| 移动到舒适位 | `/control/move_j` 话题 | 发布目标关节角度 |
| 归位 | `/move_home` 服务 | 回到机械零点 |
| 急停 | `/emergency_stop` 服务 | 立即停止所有运动 |
| 状态读取 | `/feedback/joint_states` 话题 | 实时关节状态 |
| 状态读取 | `/feedback/arm_status` 话题 | 机械臂整体状态 |

> **安全规则**：
> - 每次运动 duration_sec ≥ 2.0 秒
> - 速度百分比 ≤ 30%（MVP 阶段）
> - 运动前确认周围无障碍物
> - 保持急停通道随时可触发

#### 进阶功能详述

**④ 人体位置检测与自适应**

```
流程：
USB 摄像头（/dev/video0）
    │
    ▼
OpenCV + MediaPipe 人体姿态检测
    │
    ▼
计算用户面部在画面中的 Y 坐标比例
    │
    ├── face_y_ratio < 0.35 → 站立用户 → 目标高度 100cm
    ├── 0.35 ≤ face_y_ratio < 0.55 → 坐姿用户 → 目标高度 80cm
    └── face_y_ratio ≥ 0.55 → 轮椅用户 → 目标高度 60cm
    │
    ▼
通过 MoveIt2 IK 解算目标关节角度
    │
    ▼
平滑运动到目标位姿（duration_sec ≥ 3.0）
```

**⑤ "递"的动作**

机械臂从待机位（贴在桌面边缘）平滑伸出，将屏幕递送到用户面前：
- 轨迹：待机位 → 抬起 → 前伸 → 微调倾角 → 停在目标位置
- 全程速度缓慢（speed_percent=15%），营造"温柔递过来"的感觉

### 4.3 语音模块（仅进阶）

| 功能 | 实现方式 | 风险 |
|------|---------|------|
| 语音指令识别 | SpeechRecognition + 离线 Vosk 模型 | 火锅店环境噪音大，准确率待验证 |
| TTS 播报 | pyttsx3 离线引擎 | 机械音明显，可接受 |
| 唤醒词 | "嗨递老" → 激活语音监听 | 避免误触发 |

> **⚠ MVP 明确不含语音**：火锅店背景噪音（锅底沸腾声、聊天声、排风系统）是语音识别的天然杀手。MVP 阶段所有交互通过上位机触摸完成，这是基于真实环境的务实决策，而非技术能力不足。

#### 未来展望：语音音量自适应

老年用户常因听力下降或环境噪音而听不清语音播报。该功能让老人只需自然地说一句"我听不清"或"再说一遍"，系统自动完成两个动作：

1. **重复上一句**：从对话历史中取出最近一条 assistant 消息，重新合成语音播报
2. **提高音量**：在原有音量基础上自动提升 3-6dB，同时通过 `amixer` 或 PulseAudio 接口调整系统输出音量

设计要点：
- 关键词检测：识别"听不清""大点声""再说一遍""没听清"等自然表达，不要求精确指令
- 音量梯度：每次"听不清"递增一档（共 3 档），下一轮对话自动恢复默认音量，避免一直大声
- 视觉辅助同步：提高音量的同时，屏幕上同步放大当前播报文字（与大字版菜单风格一致）

#### 未来展望：涮菜定时提醒

老年人在火锅用餐过程中容易忘记锅里涮了多久的食物（毛肚、鹅肠等对时间敏感的食材尤甚）。该功能提供一个独立于主语音对话进程的事件中断系统：

1. **语音设定**：老人说"帮我定个 30 秒""毛肚涮 15 秒""牛肉涮 2 分钟"，系统解析时间意图
2. **屏幕倒计时**：屏幕右上角出现暖色大字倒计时（如 `🍲 毛肚 · 00:15`），不影响当前点单主界面
3. **时间到提醒**：倒计时归零时触发双通道提醒——
   - 屏幕：倒计时区域闪烁 + 大字提示"毛肚好了，可以吃了！"
   - 语音：中断当前对话（最高优先级），播放"嘀嘀嘀——毛肚时间到了，可以捞起来了"
4. **多任务并发**：支持同时设定多个定时器（如同时涮毛肚和牛肉），每个独立显示和提醒
5. **快捷取消**：老人说"取消毛肚的定时"或手动点击屏幕取消

技术要点：
- 定时引擎独立于语音对话主进程（独立线程或子进程），通过消息队列传递中断事件
- 语音中断采用优先级队列：定时提醒 > 服务员对话 > 背景音乐
- 时间解析支持自然语言："半分钟""一分半""两分钟"等老年人常用表达

### 4.4 上位机控制面板（新增）

#### 技术实现

上位机通过以下方式与 ROS2 交互：

```python
# 伪代码：上位机调用 ROS2 控制机械臂
import rclpy
from agx_arm_msgs.srv import EnableAgxArm, MoveHome, EmergencyStop

class HiDilaoController:
    def start_arm(self):
        # 1. 使能机械臂
        self.enable_client.call(EnableAgxArm.Request())
        # 2. 移动到舒适坐标
        self.publish_joint_target(COMFORT_POSE)
    
    def go_home(self):
        self.home_client.call(MoveHome.Request())
    
    def emergency_stop(self):
        self.emergency_client.call(EmergencyStop.Request())
    
    def switch_menu(self, mode: str):
        # 通过 WebSocket 通知屏幕切换页面
        self.ws.send({"action": "switch_menu", "mode": mode})
    
    def start_birthday(self):
        # 执行舞蹈序列
        for waypoint in BIRTHDAY_DANCE_SEQUENCE:
            self.publish_joint_target(waypoint["joints"])
            time.sleep(waypoint["duration"])
```
