# E1002 本地图片显示方案

E1002 可以直接通过局域网 Wi-Fi 接收 Linux 主机发送的 HTTP 指令。这个方案不需要 Home Assistant，也不需要任何云平台。E1002 收到指令后，从 SD 卡读取对应的 BMP 图片并渲染到 ePaper 屏幕，整条链路完全在本地完成。

## 整体架构

```text
Linux 主机
  |
  | 局域网 Wi-Fi HTTP 请求
  v
E1002 ESP32 HTTP 服务
  |
  | 从 SD 卡读取 BMP 图片
  v
800x480 Spectra 6 ePaper 屏幕
```

## 第一步：准备图片并放入 SD 卡

图片必须是 **24-bit BMP，800x480 像素**。颜色建议尽量只使用 Spectra 6 支持的六种颜色：黑、白、红、绿、蓝、黄。色域外的颜色会被自动映射到最近的支持色，但显示效果可能有偏差。

在 Linux 上使用 ImageMagick 转换图片：

```bash
# 安装 ImageMagick（如果没有）
sudo apt install imagemagick

# 把任意图片转成 800x480 24-bit BMP
convert 原始图片.png \
  -resize 800x480! \
  -type TrueColor \
  -define bmp:subtype=RGB \
  BMP3:menu.bmp

# 批量转换 PNG 文件
for f in *.png; do
  convert "$f" -resize 800x480! -type TrueColor BMP3:"${f%.png}.bmp"
done
```

把转换好的 BMP 文件复制到 SD 卡根目录。SD 卡建议使用 FAT32 格式，容量不超过 32 GB。

```text
SD 卡根目录/
├── menu.bmp
├── info.bmp
├── promo.bmp
└── idle.bmp
```

复制完成后，将 SD 卡插入 E1002。

## 第二步：搭建 Arduino 开发环境

在 Arduino IDE 中打开 **File -> Preferences**，添加 ESP32 board URL：

```text
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
```

然后在 Boards Manager 中安装 **esp32 by Espressif Systems**，板型选择 **XIAO_ESP32S3**。

安装 Seeed_GFX 库：

1. 从 `https://github.com/Seeed-Studio/Seeed_GFX` 下载 ZIP。
2. 在 Arduino IDE 中选择 **Sketch -> Include Library -> Add .ZIP Library**。

**Tools 菜单必须启用 OPI PSRAM**：

```text
Tools -> PSRAM -> OPI PSRAM
```

## 第三步：新建 `driver.h`

在 sketch 文件夹同级新建 `driver.h`，E1002 对应配置如下：

```cpp
#define BOARD_SCREEN_COMBO 521  // reTerminal E1002 (UC8179C)
```

## 第四步：烧录 Arduino 固件

E1002 端只需要烧录一次固件：

1. 打开 `e1002_server.ino`。
2. 将第 17-18 行附近的 Wi-Fi 名称和密码改成实际值。
3. 将 `driver.h` 放在 `e1002_server.ino` 同一个文件夹。
4. Arduino IDE 选择板型 `XIAO_ESP32S3`。
5. 设置 **Tools -> PSRAM -> OPI PSRAM**。
6. 上传固件。
7. 打开串口监视器，波特率设置为 115200，等待输出 IP 地址。

串口输出示例：

```text
WiFi connected!
IP Address: 192.168.1.88
```

## 第五步：Linux 端准备图片

安装转换工具：

```bash
sudo apt install imagemagick
```

把图片放到目标目录后运行：

```bash
./prepare_images.sh
```

将输出的 `.bmp` 文件复制到 SD 卡根目录。

## 第六步：Linux 端发送指令

### 使用 `curl`

```bash
# 切换图片
curl "http://192.168.1.88/show?img=menu"

# 查询当前状态
curl "http://192.168.1.88/status"

# 列出 SD 卡上的所有图片
curl "http://192.168.1.88/list"
```

### 使用 Python 脚本

```bash
# 先把 IP 改成 E1002 的实际地址
python3 e1002_controller.py --ip 192.168.1.88 --list
python3 e1002_controller.py --ip 192.168.1.88 menu
python3 e1002_controller.py --ip 192.168.1.88 info --wait
```

### 在自己的 Python 程序中调用

```python
from e1002_controller import E1002Controller

e = E1002Controller("192.168.1.88")
e.show("menu")   # 发出指令后立即返回，E1002 在后台刷屏
e.show("promo")  # 切换到另一张图片
```

## 关键注意事项

**关于刷屏时间：** E1002 全彩 ePaper 刷新通常需要 15-30 秒。`/show` 接口会立刻返回 `200 OK`，刷屏在后台继续进行。如果刷新过程中再次发送新指令，新指令会被排队，并在下一轮 loop 中处理。不要期待像 LCD 一样即时切换。

**关于固定 IP：** 建议在路由器中给 E1002 的 MAC 地址绑定固定 IP。这样 Linux 端脚本和 `curl` 命令可以长期使用同一个地址，不需要每次重启后重新查看 IP。E1002 的 MAC 地址会打印在串口日志中。
